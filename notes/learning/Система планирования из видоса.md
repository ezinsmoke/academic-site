# A Planning System That Saves 20+ Hours a Week

## 🎯 Ключевая идея
Создание системы планирования, которая экономит более **20 часов в неделю**, благодаря строгому распределению времени, приоритизации задач и минимизации отвлечений.

---

## 🧩 Модель работы с задачами (система автора)

1. **Инвентаризация всех задач**  
   - Все дела фиксируются сразу (никаких “в голове”).  
   - Задачи разбиваются по проектам и контекстам.

2. **Ежедневное планирование через “time blocking”**  
   - День делится на блоки по 60–90 минут.  
   - Каждый блок имеет чёткую цель (одна задача или один тип задач).  
   - В календаре нет “пустых зон” — всё имеет слот.

3. **Приоритизация по вкладу в цели**  
   - Задачи сортируются по степени влияния на долгосрочные цели.  
   - Мелкие и несущественные — группируются и выполняются в “второстепенные блоки”.

4. **Интеграция календаря и задачника**  
   - Календарь = *время исполнения*  
   - ToDo-список = *что исполняется*  
   - Ежедневная синхронизация между ними.

5. **Контроль отвлечений**  
   - Уведомления и соцсети блокируются в “глубоких блоках”.  
   - Проверка почты/мессенджеров — только в специально отведённое время.

6. **Регулярный обзор**  
   - Вечером — анализ: что сработало, что перенести.  
   - Раз в неделю — пересмотр приоритетов и проектов.

## ⚙️ Компоненты системы

| Компонент | Суть | Цель |
|------------|------|------|
| Time Blocking | Деление дня на чёткие временные блоки | Сфокусированная работа |
| Deep Work Blocks | Основные периоды максимальной концентрации | Продвижение ключевых проектов |
| Task Review | Проверка списка задач и целей | Корректировка курса |
| Priority Filter | Удаление “мелких” задач | Сохранение энергии для важного |
| Scheduled Rest | Планирование отдыха и “ничегонеделания” | Поддержка устойчивости системы |

## 🧠 Пример адаптивного внедрения (через FOSS-инструменты)

### 1. **Etar (календарь)**
- Используй для **time blocking**.  
- Разбей день на интервалы (например, 09:00–11:00 “глубокая работа”, 11:30–13:00 “учёба”, 15:00–17:00 “админ-задачи”).  
- Цветами выделяй типы активности (работа, учеба, отдых).

### 2. **Tasks (аналог ToDoist)**
- Храни **список всех задач и проектов**.  
- Добавь теги: `#deep`, `#quick`, `#routine`.  
- Каждое утро:
  - выбери 3 ключевые задачи дня;  
  - назначь им слоты в Etar.  
- Используй подзадачи для структурирования крупных проектов.

### 3. **SimpleReminder**
- Настраивай напоминания о начале “блоков” (например, “Начни Deep Work”).  
- Используй повторяющиеся напоминания: “Ежедневный обзор 21:00”, “Проверка задач на неделю — воскресенье 18:00”.

|-------|-------------|-------------|
| 08:30–09:00 | Утренний обзор / выбор 3 задач | Tasks |
| 09:00–11:00 | Глубокая работа над проектом | Etar (Deep Work Block) |
| 11:00–11:15 | Перерыв | — |
| 11:15–12:30 | Учеба / повторение материала | Etar |
| 12:30–13:00 | Проверка почты, мелкие дела | Tasks |
| 13:00–14:00 | Обед, отдых | — |
| 14:00–16:00 | Второй блок фокусированной работы | Etar |
| 16:00–16:15 | Напоминание — проверка прогресса | SimpleReminder |
| 21:00 | Вечерний обзор, перенос задач | Tasks + Etar |

Метод даёт эффект при системном применении:

- обязательный ежедневный обзор;  
- уважение к собственным “блокам фокуса”.
